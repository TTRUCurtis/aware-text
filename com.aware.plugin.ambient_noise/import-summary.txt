Manifest Merging:
-----------------

project did not explicitly turn on manifest merging. In Android Gradle
projects, manifests are always merged (meaning that contents from your
libraries' manifests will be merged into the app manifest. If you had
manually copied contents from library manifests into your app manifest
you may need to remove these for the app to build correctly.

Moved Files:
------------
Android Gradle projects use a different directory structure than ADT
Eclipse projects. Here's how the projects were restructured:

In AChartAndroid-lib:
* AndroidManifest.xml => aChartAndroidlib/src/main/AndroidManifest.xml
* assets/ => aChartAndroidlib/src/main/assets/
* libs/achartengine-1.1.0.jar => aChartAndroidlib/libs/achartengine-1.1.0.jar
* res/ => aChartAndroidlib/src/main/res/
* src/ => aChartAndroidlib/src/main/java/
In aware_framework_v2:
* AndroidManifest.xml => aware_framework_v2/src/main/AndroidManifest.xml
* assets/ => aware_framework_v2/src/main/assets/
* libs/armeabi-v7a/libiconv.so => aware_framework_v2/src/main/jniLibs/armeabi-v7a/libiconv.so
* libs/armeabi-v7a/libzbarjni.so => aware_framework_v2/src/main/jniLibs/armeabi-v7a/libzbarjni.so
* libs/armeabi/libiconv.so => aware_framework_v2/src/main/jniLibs/armeabi/libiconv.so
* libs/armeabi/libzbarjni.so => aware_framework_v2/src/main/jniLibs/armeabi/libzbarjni.so
* libs/mqtt-client-0.4.0.jar => aware_framework_v2/libs/mqtt-client-0.4.0.jar
* libs/x86/libiconv.so => aware_framework_v2/src/main/jniLibs/x86/libiconv.so
